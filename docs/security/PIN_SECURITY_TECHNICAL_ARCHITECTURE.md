## ğŸ“ API Contract Documentation

### 1. GET /api/user/pin-status

**Description:** Check if user has created a PIN

**Authentication:** Required (JWT)

**Request:**
```http
GET /api/user/pin-status HTTP/1.1
Host: localhost:5001
Authorization: Bearer <jwt_token>
```

**Response (Has PIN):**
```json
{
  "success": true,
  "data": {
    "hasPin": true
  }
}
```

**Response (No PIN):**
```json
{
  "success": true,
  "data": {
    "hasPin": false
  }
}
```

**Error Response:**
```json
{
  "success": false,
  "message": "User tidak ditemukan"
}
```

---

### 2. POST /api/user/set-pin

**Description:** Create or update user's security PIN

**Authentication:** Required (JWT)

**Request:**
```http
POST /api/user/set-pin HTTP/1.1
Host: localhost:5001
Authorization: Bearer <jwt_token>
Content-Type: application/json

{
  "pin": "123456",
  "currentPassword": "user_password"
}
```

**Validation:**
- PIN must be exactly 6 characters
- PIN must be numeric only
- currentPassword must match user's actual password

**Response (Success):**
```json
{
  "success": true,
  "message": "PIN berhasil diatur"
}
```

**Error Response (Wrong Password):**
```json
{
  "success": false,
  "message": "Password salah"
}
```

**Error Response (Invalid PIN):**
```json
{
  "success": false,
  "message": "PIN harus 6 digit angka"
}
```

---

### 3. POST /api/user/verify-pin

**Description:** Verify user's PIN for protected actions

**Authentication:** Required (JWT)

**Request:**
```http
POST /api/user/verify-pin HTTP/1.1
Host: localhost:5001
Authorization: Bearer <jwt_token>
Content-Type: application/json

{
  "pin": "123456"
}
```

**Response (Correct PIN):**
```json
{
  "success": true,
  "message": "PIN verified"
}
```

**Response (Wrong PIN):**
```json
{
  "success": false,
  "message": "PIN salah"
}
```

**Response (No PIN Set):**
```json
{
  "success": false,
  "message": "PIN belum diatur"
}
```
